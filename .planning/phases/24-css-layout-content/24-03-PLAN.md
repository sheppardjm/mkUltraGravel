**Card height fix (GravelSectors.astro):**

The root cause is that `aspect-video` on gravel card images scales image height proportional to the card's container width. Gravel cards sit in `md:col-span-2` (2/3 of the grid), so their 16:9 images are ~2x taller than KOM cards in the remaining 1/3 column.

Fix: In `src/components/GravelSectors.astro`, on the `<img>` element (currently line 36), replace `aspect-video` with `h-[180px]`. This decouples image height from container width so gravel card images render at the same fixed height regardless of column width.

Change from:
```
class="w-full aspect-video object-cover"
```
To:
```
class="w-full h-[180px] object-cover"
```

Do NOT change KomSegments.astro -- its images in the narrower column are already close to 180px with aspect-video, so the visual parity comes from gravel cards shrinking to match.

**Reset button fix (src/pages/index.astro):**

The button at lines 267-277 (id="map-reset") is invisible because it has no background color and its border (oklch 0.25) is nearly indistinguishable from the page background (oklch 0.10).

Fix: Update the button classes to add a visible background and high-contrast border. Replace the current class list:
```
class="mt-3 px-4 py-2 text-xs uppercase tracking-widest
       border border-border text-text-muted
       hover:border-accent-green hover:text-accent-green
       transition-colors cursor-pointer
       active:translate-y-px motion-reduce:active:transform-none"
```

With:
```
class="mt-6 px-4 py-2 text-xs uppercase tracking-widest
       bg-bg-surface border border-accent-green text-text-muted
       hover:bg-bg-elevated hover:text-accent-green
       transition-colors cursor-pointer
       active:translate-y-px motion-reduce:active:transform-none"
```

Changes:
- `mt-3` -> `mt-6` (more breathing room below elevation chart)
- Added `bg-bg-surface` (oklch 0.14 -- visible against bg-base 0.10)
- `border-border` -> `border-accent-green` (green border matches site brutalist style, high contrast)
- `hover:border-accent-green` removed (already the default now)
- Added `hover:bg-bg-elevated` (oklch 0.18 -- subtle hover lift)
- Kept `text-text-muted` for default state, `hover:text-accent-green` for hover

Do NOT modify the button's id, aria-label, or text content. Do NOT modify the JS click handler.

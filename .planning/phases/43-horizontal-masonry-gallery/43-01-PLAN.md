Two changes:

**1. Rewrite src/components/PhotoGallery.astro:**

Keep the existing frontmatter (fs.readFileSync of photos.json, same type assertion). Replace the template and keep the script tag unchanged.

New template structure:
- Replace the `<div id="photo-gallery" class="grid grid-cols-2 ...">` with a flex container:
  ```
  <div id="photo-gallery"
    class="flex overflow-x-auto snap-x snap-proximity overscroll-x-contain gap-1 pb-2"
    style="height: 280px;"
  >
  ```
- For each photo, compute `aspectRatio = photo.width / photo.height` and `itemWidth = Math.round(280 * aspectRatio)` and `thumbHeight = Math.round(400 / aspectRatio)`.
- Each `<a>` becomes:
  ```
  <a
    href={`/images/${photo.filename}`}
    data-pswp-width={photo.width}
    data-pswp-height={photo.height}
    class="gallery-item flex-none snap-start overflow-hidden cursor-pointer"
    style={`width: ${itemWidth}px; height: 280px;`}
    aria-label={`View route photo at mile ${photo.mi}`}
  >
  ```
- Each `<img>` gets explicit width/height attributes for CLS prevention:
  ```
  <img
    src={`/images/thumbs/${photo.filename.replace(/\.(jpg|jpeg|png|avif)$/i, '.webp')}`}
    alt={`Route photo at mile ${photo.mi}`}
    width={400}
    height={thumbHeight}
    loading="lazy"
    decoding="async"
    class="w-full h-full object-cover"
  />
  ```
- Remove the hover:scale-105 transition from images (horizontal strip items don't benefit from individual hover zoom).
- Hide scrollbar for cleaner aesthetic: add a `<style>` block after the template with:
  ```css
  #photo-gallery {
    scrollbar-width: none;
  }
  #photo-gallery::-webkit-scrollbar {
    display: none;
  }
  ```

Keep the `<script>` tag (PhotoSwipe init) exactly as-is. The existing selector `gallery: '#photo-gallery'` and `children: '.gallery-item'` will continue working because we preserved both the container id and the child class.

**2. Fix overflow on #photos section in src/pages/index.astro:**

On line 308, change:
```
<section id="photos" class="relative min-h-screen px-4 py-16 overflow-hidden border-t border-border">
```
to:
```
<section id="photos" class="relative min-h-screen px-4 py-16 overflow-x-clip border-t border-border">
```

This replaces `overflow-hidden` with `overflow-x-clip`. The `overflow-x-clip` property clips horizontal overflow without creating a scroll context, allowing the gallery's horizontal scroller to work while still containing the absolute-positioned `.tone-image` background. Do NOT use `overflow-x-hidden` (creates implicit scroll container that breaks scroll chaining).

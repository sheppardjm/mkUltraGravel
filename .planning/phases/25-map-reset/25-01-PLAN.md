**RouteMap.astro — Listen for `map:reset` and restore initial state:**

Inside `initMap()`, after the `map.fitBounds(routeLine.getBounds(), { padding: [20, 20] })` line, capture the initial bounds:

```ts
const initialBounds = routeLine.getBounds();
const initialPadding = [20, 20];
```

Then, near the bottom of `initMap()` (alongside the existing `window.addEventListener` calls that use `{ signal }`), add a `map:reset` listener:

```ts
window.addEventListener('map:reset', (() => {
  // Restore map to initial zoom/bounds
  map.fitBounds(initialBounds, { padding: initialPadding });
  // Close all open popups
  map.closePopup();
  // Reset all sector polyline styles to original
  sectorPolylines.forEach((p: any, i: number) => {
    const sector = annotations.sectors[i];
    p.setStyle({
      color: starColors[sector.stars] || '#ffffff',
      weight: 5,
      opacity: 0.9
    });
  });
  // Hide crosshair
  crosshair.setOpacity(0);
}) as EventListener, { signal });
```

Key details:
- Use `map.fitBounds` (not `flyTo`) with the same `initialBounds` and `padding: [20, 20]` that the initial load uses — this guarantees identical framing.
- `map.closePopup()` closes whichever popup is currently open (sectors, restocks, photos).
- Sector polyline reset uses the same `originalStyle` values: `weight: 5`, `opacity: 0.9`, color from `starColors[sector.stars]`.
- `crosshair.setOpacity(0)` matches the `elevation:hoverEnd` handler pattern.
- Register with `{ signal }` on the existing AbortController for cleanup consistency.

**ElevationProfile.astro — Listen for `map:reset` and restore annotation bands:**

Near the bottom of `initElevation()`, alongside the existing `map:sectorHover` and `map:sectorClick` listeners (which also use `{ signal }`), add:

```ts
// Listen for map:reset — restore all sector bands to default opacity
window.addEventListener('map:reset', (() => {
  const annots = (chartInstance as any).options.plugins.annotation.annotations;
  Object.keys(annots).forEach(key => {
    const annot = annots[key];
    if (annot._baseColor) {
      annot.backgroundColor = annot._baseColor + '22';  // default ~13% opacity
      annot.borderColor = annot._baseColor + '66';       // default ~40% opacity
    }
  });
  (chartInstance as any).update('none');
}) as EventListener, { signal });
```

Key details:
- The default sector annotation opacity values are `_baseColor + '22'` (background) and `_baseColor + '66'` (border) — these match the initial values set in the `annotationBoxes` construction loop.
- `update('none')` skips animation, same pattern used by the existing `map:sectorHover` and `map:sectorClick` handlers.
- KOM annotations don't have `_baseColor` (by design — see 22-01 decisions), so the `if (annot._baseColor)` guard naturally skips them. KOM annotations don't change on sector interactions, so they need no reset.
- Register with `{ signal }` for cleanup consistency.

<objective>
Prepare the git repository for Cloudflare Pages deployment by adding .gitignore, .node-version, committing all source assets (images/ and GPX), and pushing to GitHub.

Purpose: Cloudflare Pages clones the git repo to build. Without source images and the GPX file in git, the prebuild pipeline fails. Without .gitignore, node_modules/ and dist/ would bloat the repo.

Output: A clean git repo on GitHub with all source assets committed, build artifacts excluded, and Node version pinned.
</objective>

<task type="auto">
  <name>Task 1: Create .gitignore and .node-version</name>
  <files>.gitignore, .node-version</files>
  <action>
Create `.gitignore` at the repo root with these entries:

```
# Dependencies
node_modules/

# Build output
dist/

# Astro generated
.astro/

# Generated images (rebuilt from images/ by prebuild)

# OS files
.DS_Store

# Environment
.env
.env.*

# Playwright MCP
.playwright-mcp/
```

Create `.node-version` at the repo root with content:
```
22
```

Important notes:
- `public/images/` MUST be in .gitignore because it is generated by the prebuild step (copies from `images/` source dir). Only `images/` (source) should be tracked.
- `public/data/` and `public/tone/` are already tracked in git and should NOT be in .gitignore -- they contain committed data files and tone images.
- Do NOT add `images/` to .gitignore -- that is the source directory that must be committed.
- Do NOT add `public/mk-ultra.gpx` to .gitignore -- it is already tracked.
- `.node-version` uses just `22` (not `22.22.2`) because Cloudflare Pages may not have the exact Volta-pinned version.
  </action>
  <verify>
Run `cat .gitignore` and confirm it contains node_modules/, dist/, .astro/, public/images/, .DS_Store entries.
Run `cat .node-version` and confirm it contains `22`.
Run `git status` and confirm .DS_Store files, node_modules/, dist/, .astro/, public/images/ do NOT appear as untracked (they are now ignored).
  </verify>
  <done>.gitignore and .node-version exist with correct content. Previously untracked build artifacts are now ignored by git.</done>
</task>

Stage and commit in this order:

   git add "MK Ultra.gpx"

   ```
   npm run build
   ```
   Build MUST succeed with no errors. The prebuild step reads from `images/` and `MK Ultra.gpx`, both now committed.

5. Push all commits to GitHub:
   ```
   git push origin main
   ```

Important:
- Do NOT use `git add .` or `git add -A` -- only add specific paths to avoid accidentally including sensitive files or build artifacts.
- The `images/` directory is ~33MB with 50 files. This is acceptable for a static site's source assets.
- If `npm run build` fails, diagnose and fix before pushing. The build must work locally before it can work on Cloudflare.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Configure custom domain (if option-a selected)</name>
  <action>
Skip this task if option-b was selected in Task 1.

The user must configure the custom domain through the Cloudflare dashboard because it requires DNS configuration that depends on whether the domain's DNS is already on Cloudflare or with an external registrar.
  </action>
  <instructions>
**If your domain's DNS is already on Cloudflare:**

1. Go to Cloudflare Dashboard > Workers & Pages > your project > Custom domains
2. Click **Set up a custom domain**
3. Enter your domain (e.g., `mkultragravel.com`)
4. Cloudflare auto-creates the CNAME record -- click **Activate domain**
5. SSL certificate auto-provisions (takes a few minutes)
6. Optionally add `www` subdomain too and set up a redirect rule

**If your domain's DNS is with an external registrar (GoDaddy, Namecheap, etc.):**

1. Go to Cloudflare Dashboard > Workers & Pages > your project > Custom domains
2. Click **Set up a custom domain**
3. Enter your domain
4. Cloudflare will show you a CNAME record to add at your registrar:
   - Type: `CNAME`
   - Name: `@` (or your subdomain)
   - Target: `mkultragravel.pages.dev` (your pages.dev URL)
5. Add this CNAME at your domain registrar's DNS settings
6. Return to Cloudflare and click **Check DNS** to verify
7. SSL certificate provisions after DNS propagates

**After HTTPS is active**, paste the custom domain URL here (e.g., https://mkultragravel.com).
  </instructions>
  <resume-signal>Paste the custom domain URL with HTTPS, or type "skipped" if option-b was chosen.</resume-signal>

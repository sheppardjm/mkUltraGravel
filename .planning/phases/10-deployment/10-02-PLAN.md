<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Cloudflare Pages project via dashboard</name>
  <action>
The user must create the Cloudflare Pages project through the dashboard because the initial git-connected setup requires OAuth authorization between Cloudflare and GitHub.

**Prerequisites:**
- Cloudflare account (free tier is sufficient)
- GitHub repo pushed to origin/main (completed in 10-01)
  </action>
  <instructions>
**Steps to create the Cloudflare Pages project:**

1. Go to https://dash.cloudflare.com
2. Navigate to **Workers & Pages** in the left sidebar
3. Click **Create** > select **Pages** tab > **Connect to Git**
4. Authorize Cloudflare to access your GitHub account (if not already)
5. Select the **mkUltraGravel** repository
6. Configure build settings:
   - **Project name:** `mkultragravel` (or your preference -- this becomes the *.pages.dev subdomain)
   - **Production branch:** `main`
   - **Framework preset:** Select **Astro** (if available) or leave as **None**
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
   - **Root directory:** (leave empty)
7. Under **Environment variables**, add:
   - Variable name: `NODE_VERSION`
   - Value: `22`
8. Click **Save and Deploy**

**Wait for the first build to complete.** This may take 2-5 minutes. The build log should show:
- `npm install` succeeding (including sharp)
- `prebuild` running (generate-data.js processing images and GPX)
- `astro build` producing output in `dist/`

**After the build completes**, copy the deployment URL (e.g., `https://mkultragravel.pages.dev`) and paste it here.
  </instructions>
  <resume-signal>Paste the deployment URL (e.g., https://mkultragravel.pages.dev) or describe any build errors from the CF Pages build log.</resume-signal>
</task>

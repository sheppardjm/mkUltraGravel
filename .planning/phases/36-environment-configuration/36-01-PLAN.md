  <action>The user must complete all of the following manual steps in order. Claude cannot automate these because they require authenticated dashboard access.</action>
  <instructions>
Complete these steps in order. Check each off as you go.

**Step 1: Create Netlify Build Hook**
1. Go to Netlify dashboard > your site > Project configuration > Build & deploy > Continuous deployment > Build hooks
2. Click "Add build hook"
3. Name: `Athlete Result Submission`
4. Branch: `main`
5. Click "Save" and copy the generated URL (format: `https://api.netlify.com/build_hooks/XXXXXXXX`)
6. Keep this URL for Step 3

**Step 2: Verify/Create GitHub PAT**
1. Go to github.com/settings/tokens (Fine-grained tokens tab)
2. Check if you have an existing PAT for mkUltraGravel:
   - If yes: verify it has Contents: Read and Write permission, and expiry is either "No expiration" or after June 7, 2026
   - If no: click "Generate new token", name it `mkUltraGravel Netlify Functions`, set expiry to "No expiration" or a date after June 7 2026, select repository `Sheppardjm/mkUltraGravel`, under Repository permissions set Contents to "Read and write", click "Generate token"
3. Copy the token value (you will need it for Step 3)

**Step 3: Set all 8 Netlify Environment Variables**
1. Go to Netlify dashboard > your site > Project configuration > Environment variables
2. Add each variable (click "Add a variable" > "Add a single variable"):

| Variable | Value | Scope |
|----------|-------|-------|
| `STRAVA_CLIENT_ID` | From strava.com/settings/api "Client ID" | All (or Functions) |
| `STRAVA_CLIENT_SECRET` | From strava.com/settings/api "Client Secret" | All (or Functions) |
| `STRAVA_REDIRECT_URI` | `https://mkultragravel.netlify.app/.netlify/functions/strava-callback` | All (or Functions) |
| `STRAVA_VERIFY_TOKEN` | The hex token generated in Task 1 | All (or Functions) |
| `GITHUB_TOKEN` | The PAT from Step 2 | All (or Functions) |
| `GITHUB_OWNER` | `Sheppardjm` | All (or Functions) |
| `GITHUB_REPO` | `mkUltraGravel` | All (or Functions) |
| `NETLIFY_BUILD_HOOK` | The build hook URL from Step 1 | All (or Functions) |

Note: On Netlify's free plan, scope defaults to "All" which is acceptable. On Pro plans, "Functions" scope is sufficient.

**Step 4: Set Strava Authorization Callback Domain**
1. Go to strava.com/settings/api
2. Set "Authorization Callback Domain" to: `mkultragravel.netlify.app`
   - Domain only — no `https://`, no trailing path
3. Save

**Step 5: Confirm Node.js version in deploy log**
1. Go to Netlify dashboard > your site > Deploys
2. Click the most recent deploy
3. In the deploy log, search for "Node version" or "node"
4. Confirm it shows `v22.x.x` (any 22.x is acceptable)

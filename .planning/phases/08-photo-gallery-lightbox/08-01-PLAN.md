Create the WebP thumbnail generation pipeline and add image dimensions to photos.json.

Purpose: The photo gallery needs optimized thumbnails (~20-40KB WebP instead of ~380KB JPEG) for fast grid loading, and PhotoSwipe requires per-image width/height for correct lightbox layout. This plan builds the data foundation that the gallery component (Plan 02) consumes.

Output: `scripts/generate-thumbnails.js`, updated `generate-data.js` coordinator, sharp as explicit devDependency, and photos.json enriched with width/height fields.
</objective>

<task type="auto">
  <name>Task 1: Install sharp and create generate-thumbnails.js</name>
  <files>package.json, scripts/generate-thumbnails.js</files>
  <action>
1. Add sharp as an explicit devDependency:
   ```
   npm install --save-dev --include=optional sharp
   ```
   This ensures the native binary is installed for the current platform (darwin-arm64 or darwin-x64). sharp may already be in node_modules as an Astro peer dep, but making it explicit prevents breakage.

2. Create `scripts/generate-thumbnails.js` (CommonJS, matching existing scripts):

   The script must:
   - Read `public/data/photos.json` (the output of match-photos.js)
   - For each photo entry, read the full-size image from `public/images/{filename}`
   - Use `sharp(srcPath).metadata()` to get actual width and height of the original
   - Resize to 600px wide (withoutEnlargement: true), convert to WebP (quality: 75, effort: 4)
   - Write to `public/images/thumbs/{basename}.webp` (same basename, .webp extension)
   - Be idempotent: skip if the thumb already exists (fs.existsSync check)
   - After processing all photos, update each entry in the photos array with `width` and `height` fields (original image dimensions, NOT thumbnail dimensions — PhotoSwipe needs full-size dims)
   - Write the enriched array back to `public/data/photos.json`
   - Log count of thumbnails generated and skipped

   Key details:
   - Use `const sharp = require('sharp')` (CommonJS like all other scripts)
   - Use `path.parse(filename).name + '.webp'` for thumbnail filename derivation
   - Create `public/images/thumbs/` directory with `fs.mkdirSync(destDir, { recursive: true })`
   - Most images are 1200x1600 (portrait), 4 are landscape (1600x1200), 1 is unusual (1133x816) — do NOT hardcode dimensions, always read from sharp metadata
   - The script processes sequentially (for loop with await) — 33 images is fast enough, no need for parallelism
   - Export the main function so generate-data.js can call it, OR make it runnable standalone with `if (require.main === module)` pattern

   Example thumbnail path derivation:
   `abc-1536x2048.jpg` -> `public/images/thumbs/abc-1536x2048.webp`
  </action>

- `node -e "const p = require('./public/data/photos.json'); console.log(p[0].width, p[0].height, p.every(x => x.width && x.height))"` prints dimensions and `true`
  </verify>

### Verified Behaviors Summary

**Grid:**
- `PhotoGallery.astro` reads `public/data/photos.json` at Astro build time via `fs.readFileSync`
- Renders 33 `<a class="gallery-item">` elements in a responsive grid: 2-col mobile, 3-col tablet, 4-col desktop
- Each thumbnail uses `aspect-[3/4] overflow-hidden` with `object-cover` — consistent with dark brutalist aesthetic
- First 8 thumbnails load `eager`, remaining 25 load `lazy`

**Thumbnails (optimization):**
- 33 WebP files at 600px wide, ranging from 44KB to 173KB
- Generated by `sharp` pipeline from full-size JPEGs (which are 1200–2048px wide)
- Pipeline is idempotent and integrated into `generate-data.js` coordinator

**Lightbox:**
- Standard PhotoSwipe 5 `gallery`/`children` selector pattern (not the broken `dataSource+loadAndOpen` approach)
- `href` points to full-size JPEG; `data-pswp-width`/`data-pswp-height` store original dimensions
- PhotoSwipe 5 defaults provide: close button (X), Escape key, click-outside-image dismiss, arrow key navigation
- No config overrides that would disable any close mechanism
- `.pswp__img { max-width: none }` in `@layer components` (higher priority than `@layer photoswipe`) — fixes Tailwind v4 preflight conflict that caused 0px-width images

<task type="auto">
  <name>Task 1: Add OG, Twitter Card, and canonical meta tags to BaseLayout.astro</name>
  <files>src/layouts/BaseLayout.astro</files>
  <action>
Edit `src/layouts/BaseLayout.astro` to add 11 meta tags in the `<head>` section. Add them after the existing `<meta name="description">` tag and before the favicon link.

In the frontmatter (between the `---` fences), add two computed values after the existing destructured props:

```
const canonicalURL = new URL(Astro.url.pathname, Astro.site);
const ogImageURL = new URL('/og-image.jpg', Astro.site);
```

In the `<head>` section, add these 11 tags (after `<meta name="description">`, before `<link rel="icon">`):

```html
<link rel="canonical" href={canonicalURL} />

<!-- Open Graph -->
<meta property="og:type" content="website" />
<meta property="og:site_name" content="MK Ultra Gravel" />
<meta property="og:url" content={canonicalURL} />
<meta property="og:title" content={title} />
<meta property="og:description" content={description} />
<meta property="og:image" content={ogImageURL} />

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content={title} />
<meta name="twitter:description" content={description} />
<meta name="twitter:image" content={ogImageURL} />
```

IMPORTANT:
- Use `Astro.site` (not `Astro.url`) as the base for ogImageURL to ensure absolute URLs in production, not localhost in dev.
- Use `new URL(Astro.url.pathname, Astro.site)` for canonicalURL to get path-specific canonical URLs per page.
- Do NOT use the `<slot name="head" />` for these tags -- they belong directly in the layout since they use props already available.
- Do NOT install any library. This is raw HTML meta tags.
- Do NOT change any existing tags or elements in the file.

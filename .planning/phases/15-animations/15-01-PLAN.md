  <action>
In `src/styles/global.css`, inside the existing `@layer components { ... }` block (after the `.sector-badge span` rule, before the closing `}`), add the `.card-hover` class with the ::after pseudo-element shadow trick:

```css
  /* Card hover — brutalist hard-shift shadow via ::after opacity trick (compositor-safe) */
  .card-hover {
    position: relative;
  }
  .card-hover::after {
    content: "";
    position: absolute;
    inset: 0;
    z-index: -1;
    box-shadow: 4px 4px 0 0 var(--color-accent-green);
    opacity: 0;
    transition: opacity 0ms step-start;
  }
  .card-hover:hover::after {
    opacity: 1;
  }
  @media (prefers-reduced-motion: reduce) {
    .card-hover::after {
      transition: none;
    }
    .card-hover:hover::after {
      opacity: 0;
    }
  }
```

**Important constraints:**
- The `::after` pseudo-element trick avoids direct `transition-shadow` which is compositor-unsafe and causes repaints.
- `transition: opacity 0ms step-start` makes the shadow appear instantly (hard snap, not fade).
- The `z-index: -1` on `::after` ensures the shadow sits behind the card content.
- Under `prefers-reduced-motion: reduce`, the shadow never appears (opacity stays 0).
- Do NOT add any `transition-transform` or `hover:translate` to the `.card-hover` class — keep it shadow-only. The transform shift is NOT needed; shadow alone is the brutalist effect.
- The `.classified-border` class already has `position: relative` — `.card-hover` reinforces it (no conflict).
  </action>

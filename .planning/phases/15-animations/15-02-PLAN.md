**Step 2: Add data-reveal utility rules after the `@layer components { ... }` block.**

After the `@layer components` closing brace, add a new `@layer utilities` block:

```css
@layer utilities {
  /* Scroll-reveal: hidden state only when JS has confirmed it is running.
     The IntersectionObserver script adds data-reveal-ready to <html>.
     Without JS, elements remain fully visible (no-JS safe). */
  [data-reveal-ready] [data-reveal] {
    opacity: 0;
  }
  [data-reveal-ready] [data-reveal].is-visible {
    animation: var(--animate-reveal);
  }
  @media (prefers-reduced-motion: reduce) {
    [data-reveal-ready] [data-reveal] {
      opacity: 1 !important;
      animation: none !important;
    }
  }
}
```

**Key constraints:**
- The `[data-reveal-ready]` prefix on `<html>` ensures that `opacity: 0` is ONLY applied when JS is running. If JS fails to load, the `data-reveal-ready` attribute never gets added and elements stay fully visible.
- The animation only uses `opacity` and `transform` — both are GPU-compositor-safe. No layout properties animated.
- `0.35s ease-out` is appropriate for entrance — this is NOT a hover transition, so ease-out (not brutalist step-start) is correct for scroll reveals. The brutalist constraint applies to hover/click states, not entrance animations.
- The `@layer utilities` declaration ensures these rules have the highest specificity in the cascade layer order (`@layer leaflet, photoswipe, base, components, utilities`).
- Do NOT touch the existing `@keyframes noise-anim` or `@keyframes noise-anim-2` in `index.astro` — those are scoped to the hero glitch effect.

**Add centralized IntersectionObserver script** at the bottom of index.astro, just before the closing `</BaseLayout>` tag (after `</main>` closing tag, before `</BaseLayout>`):

```astro
<script>
  // Scroll-reveal: IntersectionObserver adds .is-visible to [data-reveal] elements
  // Centralized in index.astro — covers all [data-reveal] elements across all components
  function initReveal() {
    // Mark <html> as JS-ready so CSS applies opacity: 0 initial state
    document.documentElement.setAttribute("data-reveal-ready", "");

    // Respect prefers-reduced-motion — skip observer entirely
    if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) {
      return;
    }

    const revealObserver = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            entry.target.classList.add("is-visible");
            revealObserver.unobserve(entry.target);
          }
        });
      },
      {
        rootMargin: "0px 0px -40px 0px",
        threshold: 0.05,
      }
    );

    document.querySelectorAll("[data-reveal]").forEach((el) => {
      revealObserver.observe(el);
    });
  }

  // Support Astro View Transitions
  document.addEventListener("astro:page-load", initReveal);
  // Fallback for non-View-Transitions pages
  if (!document.querySelector("[data-astro-transition]")) {
    initReveal();
  }
</script>
```

Add a `<script>` block at the bottom of `results.astro` (before the closing `</BaseLayout>` tag) with vanilla JS tab switching logic.

The tab system must support TWO independent tab groups: "gravel" and "kom". Each group has its own set of tab buttons and panels. Use `data-section` and `data-gender` attributes to scope the tab switching.

```javascript
// Tab switching — scoped by data-section to support multiple independent tab groups
const tabButtons = document.querySelectorAll('[role="tab"]');
const tabPanels = document.querySelectorAll('[role="tabpanel"]');

tabButtons.forEach(btn => {
  btn.addEventListener('click', () => {
    const section = btn.dataset.section;
    const gender = btn.dataset.gender;

    // Update tabs within this section only
    tabButtons.forEach(t => {
      if (t.dataset.section === section) {
        const isActive = t.dataset.gender === gender;
        t.setAttribute('aria-selected', String(isActive));
        // Toggle active/inactive styling
        if (isActive) {
          t.style.backgroundColor = 'var(--color-bg-elevated)';
          t.style.color = 'var(--color-accent-white)';
        } else {
          t.style.backgroundColor = 'transparent';
          t.style.color = 'var(--color-text-muted)';
        }
      }
    });

    // Show/hide panels within this section only
    tabPanels.forEach(p => {
      if (p.dataset.section === section) {
        p.hidden = p.dataset.gender !== gender;
      }
    });
  });
});
```

Only include this script block conditionally — wrap it in `{hasResults && ( <script>...</script> )}` so it's omitted in the empty state.

**Important:** Do NOT use TypeScript type assertions (e.g. `as HTMLElement`) in this script — use plain JavaScript to match the pattern in submit.astro which uses `as` only because Astro processes it. Actually, Astro DOES process inline `<script>` tags as TypeScript, so `as` casts are fine. Follow the exact pattern from submit.astro.

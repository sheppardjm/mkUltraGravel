<task type="auto">
  <name>Task 1: Add global scrollbar theme to global.css</name>
  <files>src/styles/global.css</files>
  <action>
Add scrollbar theming declarations in two places within global.css:

1. Inside the existing `@layer base` block, add `scrollbar-color` and `scrollbar-width` to the existing `html` rule (NOT a new `:root` block — use the existing `html` selector for consistency with the file's pattern):

```css
html {
  /* existing properties stay */
  scrollbar-color: var(--color-accent-green) var(--color-bg-surface);
  scrollbar-width: thin;
}
```

This uses the standard CSS Scrollbars Level 1 properties (Baseline 2025). `scrollbar-color` syntax is `thumb track` — accent-green thumb on dark surface track. These properties inherit to all child elements, covering SCROLL-01 (main vertical), SCROLL-02 (gallery horizontal), and SCROLL-03 (any other scrollable containers).

2. AFTER the closing `}` of `@layer components` (line 265) and BEFORE `@keyframes escher-drift` (line 267), add the WebKit fallback block:

```css
/* WebKit scrollbar fallback — Chrome <121, macOS Safari <18 */
@supports selector(::-webkit-scrollbar) {
  ::-webkit-scrollbar {
    width: 8px;
    height: 8px;
  }
  ::-webkit-scrollbar-track {
    background: var(--color-bg-surface);
  }
  ::-webkit-scrollbar-thumb {
    background: var(--color-accent-green);
  }
  ::-webkit-scrollbar-thumb:hover {
    background: var(--color-accent-white);
  }
  ::-webkit-scrollbar-corner {
    background: var(--color-bg-base);
  }
}
```

The `@supports selector(::-webkit-scrollbar)` guard is the MDN-recommended pattern to prevent conflicts. Place this OUTSIDE any @layer so it applies globally.

Do NOT place the WebKit fallback inside `@layer base` or `@layer components` — it must be in the ungrouped stylesheet scope so pseudo-element selectors are not layer-scoped.
  </action>

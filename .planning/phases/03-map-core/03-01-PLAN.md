<task type="auto">
  <name>Task 2: Create RouteMap.astro component and wire into index.astro</name>
  <files>src/components/RouteMap.astro, src/pages/index.astro</files>
  <action>
1. Create `src/components/RouteMap.astro` with:

   **HTML:** A single `<div id="map" class="route-map"></div>` container.

   **Style block:** Give `.route-map` an explicit height of `60vh` with `min-height: 400px` and `width: 100%`. Leaflet requires non-zero height before initialization (Pitfall 4 in research). Also add `z-index: 0` and `position: relative` so it sits below the grain overlay (z-index 9999).

   **Script block (top-level await, all client-side):**

   a. Dynamic-import Leaflet to avoid SSR "window is undefined" error (Pitfall 1):
      ```js
      const L = (await import('leaflet')).default;
      ```
      Do NOT use static `import L from 'leaflet'` at module top level.

   b. Dynamic-import and wire GestureHandling BEFORE map init (Pitfall 6):
      ```js
      const { GestureHandling } = await import('leaflet-gesture-handling');
      L.Map.addInitHook('addHandler', 'gestureHandling', GestureHandling);
      ```

   c. Initialize the map with gestureHandling enabled:
      ```js
      const map = L.map('map', { gestureHandling: true });
      ```

   d. Add Carto Dark Matter tile layer (no API key required):
      ```js
      L.tileLayer('https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png', {
        attribution: '&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors &copy; <a href="https://carto.com/attributions">CARTO</a>',
        subdomains: 'abcd',
        maxZoom: 20
      }).addTo(map);
      ```

   e. Fetch route-data.json and render the GPX polyline:
      ```js
      const routeData = await fetch('/data/route-data.json').then(r => r.json());
      const latlngs = routeData.map(pt => [pt.lat, pt.lon]);
      const routeLine = L.polyline(latlngs, {
        color: '#d4d4d4',
        weight: 3,
        opacity: 0.7,
        smoothFactor: 1
      }).addTo(map);
      ```
      Use `#d4d4d4` (close to accent-white token) for the route line — subtle on dark tiles.

   f. Fit map bounds to the route polyline:
      ```js
      map.fitBounds(routeLine.getBounds(), { padding: [20, 20] });
      ```

2. In `src/pages/index.astro`:

   a. Add import at top of frontmatter:
      ```astro
      import RouteMap from "../components/RouteMap.astro";
      ```

   b. Replace the #route section content. Keep the section element, h2, and tone image. Remove the placeholder `<p>` text. Add `<RouteMap />` inside the `relative z-10` div, below the h2:
      ```astro
      <div class="relative z-10">
        <h2 class="text-3xl md:text-5xl mb-8">The Route</h2>
        <RouteMap />
      </div>
      ```

   IMPORTANT: The RouteMap is a vanilla .astro component with a `<script>` tag — it does NOT use `client:load` (that's for framework islands only). Astro script tags in .astro components are automatically bundled and run client-side.

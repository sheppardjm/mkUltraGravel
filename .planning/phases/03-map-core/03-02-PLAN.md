   ```js
   const [routeData, annotations] = await Promise.all([
     fetch('/data/route-data.json').then(r => r.json()),

   ]);
   ```

2. **Render sector overlays.** Define a star-to-color mapping using the design system's brutalist palette feel:
   ```js
   const starColors = {
     1: '#888888',   // light gravel — muted gray
     2: '#aaaaaa',   // moderate — lighter gray
     3: '#f5a623',   // amber — getting rough
     4: '#e86d1f',   // orange — hard
     5: '#c0392b'    // red — brutal
   };
   ```

   For each sector in `annotations.sectors`, create a polyline from `sector.track` (array of `{lat, lon}` objects) and bind a popup:
   ```js
   annotations.sectors.forEach(sector => {
     const latlngs = sector.track.map(pt => [pt.lat, pt.lon]);
     L.polyline(latlngs, {
       color: starColors[sector.stars] || '#ffffff',
       weight: 5,
       opacity: 0.9
     })
     .bindPopup(
       `<strong>${sector.name}</strong><br>` +
       `${'★'.repeat(sector.stars)}${'☆'.repeat(5 - sector.stars)}<br>` +
       `${sector.lengthMi.toFixed(1)} mi`
     )
     .addTo(map);
   });
   ```

3. **Render KOM overlays.** Use chartreuse green (`#7fff00`) with dashed lines to visually distinguish from sector colors:
   ```js
   annotations.kom.forEach(kom => {
     const latlngs = kom.track.map(pt => [pt.lat, pt.lon]);
     L.polyline(latlngs, {
       color: '#7fff00',
       weight: 6,
       opacity: 0.85,
       dashArray: '8, 4'
     })
     .bindPopup(
       `<strong>${kom.name}</strong><br>` +
       `Grade: ${kom.grade}%<br>` +
       `Elevation: ${kom.elevFt} ft<br>` +
       `${kom.lengthMi.toFixed(1)} mi`
     )
     .addTo(map);
   });
   ```
   Note: Use `kom.grade` and `kom.elevFt` — these are the actual field names (NOT `gradient`/`elevGain` as the research incorrectly stated). The values are real numbers (e.g., grade=6.4, elevFt=236), not null.

After the KOM overlay code in the `<script>` block, add restock point markers.

Use `L.divIcon` (not default markers — Pitfall 2: default icon PNGs break in Vite production builds):

```js
annotations.restock.forEach(stop => {
  const icon = L.divIcon({
    className: 'restock-marker',
    html: '<div style="width:12px;height:12px;background:#22d3ee;border:2px solid #fff;border-radius:50%;"></div>',
    iconSize: [16, 16],
    iconAnchor: [8, 8],
    popupAnchor: [0, -10]
  });

  L.marker([stop.lat, stop.lon], { icon })
    .bindPopup(`<strong>${stop.name}</strong><br>Mile ${stop.mi}`)
    .addTo(map);
});
```

Notes:
- Use `stop.lat` and `stop.lon` directly — restock entries have these fields (verified; research Pitfall 5 was wrong).
- Use `stop.mi` for the mile marker display (the rounded value from the source data).
- Cyan (`#22d3ee`) for restock markers — distinct from red/orange sectors and green KOMs.
- Inline styles on the divIcon HTML avoids needing a separate CSS class (the marker is simple enough).
- White border gives visibility on dark tiles.

Also add a minimal CSS rule for the restock-marker class in the `<style>` block to prevent Leaflet's default divIcon styling from adding unwanted background/border:

```css
:global(.restock-marker) {
  background: transparent !important;
  border: none !important;
}
```

Use `:global()` because Leaflet appends the marker to the DOM outside the component's scoped style boundary.

1. In `src/styles/global.css`, add the following rules inside the existing `@layer components { ... }` block (after the existing component styles like `.tone-image`):

   ```css
   /* Leaflet dark theme overrides */
   .leaflet-popup.dark-popup .leaflet-popup-content-wrapper {
     background: oklch(0.18 0.01 250);
     color: oklch(0.85 0.01 90);
     border: 1px solid oklch(0.25 0.01 250);
     border-radius: 2px;
     font-family: var(--font-mono);
     font-size: 0.75rem;
     box-shadow: 0 2px 8px rgba(0, 0, 0, 0.5);
   }

   .leaflet-popup.dark-popup .leaflet-popup-tip {
     background: oklch(0.18 0.01 250);
     border: 1px solid oklch(0.25 0.01 250);
   }

   .leaflet-popup.dark-popup .leaflet-popup-close-button {
     color: oklch(0.55 0.01 90);
   }
   .leaflet-popup.dark-popup .leaflet-popup-close-button:hover {
     color: oklch(0.85 0.01 90);
   }
   ```

   Use the raw oklch values matching the design tokens (bg-elevated, text-body, border, text-muted). Using raw values instead of var() references because Leaflet injects popups outside the Astro component scope where CSS custom properties from @theme may not resolve in all contexts.

2. Also add attribution and zoom control styling to match the dark theme:

   ```css
   .leaflet-control-attribution {
     background: oklch(0.10 0.01 250 / 0.8) !important;
     color: oklch(0.55 0.01 90) !important;
     font-size: 0.6rem !important;
   }
   .leaflet-control-attribution a {
     color: oklch(0.55 0.01 90) !important;
   }

   .leaflet-control-zoom a {
     background: oklch(0.18 0.01 250) !important;
     color: oklch(0.85 0.01 90) !important;
     border-color: oklch(0.25 0.01 250) !important;
   }
   .leaflet-control-zoom a:hover {
     background: oklch(0.25 0.01 250) !important;
   }
   ```

   The `!important` is necessary here because Leaflet's own CSS (in @layer leaflet) would normally win on specificity within its layer, but these component-layer overrides need to ensure they take effect. Since @layer components is declared after @layer leaflet in the cascade order, they should win — but Leaflet inlines some styles, so !important is the safe approach.

3. In `src/components/RouteMap.astro`, update ALL existing `.bindPopup()` calls to pass the `className` option. Change every `.bindPopup('...')` to `.bindPopup('...', { className: 'dark-popup' })`.

   There should be three sets of bindPopup calls to update:
   - Sector popups (inside the `annotations.sectors.forEach` loop)
   - KOM popups (inside the `annotations.kom.forEach` loop)
   - Restock popups (inside the `annotations.restock.forEach` loop)

4. Also add styling for the gesture handling overlay text to match the dark theme:

   ```css
   .leaflet-gesture-handling-touch-warning,
   .leaflet-gesture-handling-scroll-warning {
     color: oklch(0.85 0.01 90);
     font-family: var(--font-mono);
     font-size: 0.875rem;
   }
   ```

   sharp('public/tone/CIA-MKULTRA-IG_Page_01.jpg')
     .webp({ quality: 80 })

   **Important:** The hero image is a grayscale CIA document — it compresses extremely well in WebP. Quality 80 should produce excellent results.

2. Add the hero image conversion to `scripts/generate-data.js` so it runs automatically on `npm run build` / `npm run dev`. Add it after the thumbnail generation step (or alongside it). Pattern:

   ```javascript
   // Convert hero image to WebP for LCP optimization
   const heroSrc = path.join(__dirname, '..', 'public/tone/CIA-MKULTRA-IG_Page_01.jpg');
   const heroDest = path.join(__dirname, '..', 'public/tone/CIA-MKULTRA-IG_Page_01.webp');
   if (!fs.existsSync(heroDest)) {
     const sharp = require('sharp');
     await sharp(heroSrc).webp({ quality: 80 }).toFile(heroDest);
     console.log('Hero image converted to WebP');
   } else {
     console.log('Hero WebP already exists, skipping');
   }
   ```

   Note: generate-data.js uses execSync (sync pattern). If sharp requires async, create a small `scripts/convert-hero.js` and call it via execSync, matching the generate-thumbnails.js pattern. Perform a systematic audit of all CSS transitions and animations in the codebase to confirm none use layout-triggering properties.

Search for all instances of `transition-`, `animation`, `@keyframes`, `transform`, `hover:` in:
- `src/styles/global.css`
- `src/pages/index.astro`
- `src/components/*.astro`

For each found animation/transition, classify the animated property:

**Compositor-safe (PASS):** `transform`, `opacity`
**Paint-only (ACCEPTABLE):** `background-color`, `color`, `box-shadow`, `border-color`
**Layout-triggering (FAIL):** `width`, `height`, `top`, `left`, `right`, `bottom`, `margin`, `padding`, `border-width`, `font-size`

| Location | Animation | Property | Verdict |
|----------|-----------|----------|---------|

| ElevationProfile.astro | animation: false (Chart.js) | None | PASS |

Confirm this matches reality. If any unexpected animations are found that animate layout-triggering properties, report them clearly — they would need to be fixed.

**Important:** Also check for any Leaflet or PhotoSwipe CSS that might animate layout properties. The library CSS is imported in global.css — search the node_modules source or the imported layer for `@keyframes` or `transition` rules that animate `width`, `height`, `top`, `left`.

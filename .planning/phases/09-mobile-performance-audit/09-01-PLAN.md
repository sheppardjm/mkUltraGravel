<task type="auto">
  <name>Task 1: Calculate WCAG AA contrast ratios for all oklch token pairs and identify failures</name>
  <files>src/styles/global.css</files>
  <action>
Write a Node.js script (temporary, run once) that converts each oklch token pair to sRGB and calculates the WCAG 2.1 relative luminance contrast ratio. This eliminates reliance on external web tools and gives exact numbers.

The oklch-to-sRGB conversion requires:
1. oklch -> oklab (polar to rectangular: a = C * cos(h), b = C * sin(h))
2. oklab -> linear sRGB (via the inverse of the oklab matrix)
3. linear sRGB -> sRGB (gamma encoding)
4. sRGB -> WCAG relative luminance (0.2126*R + 0.7152*G + 0.0722*B with linearization)

Token pairs to audit (from research):

| # | Foreground | Background | Context | Threshold |
|---|-----------|-----------|---------|-----------|
| 1 | oklch(0.85 0.01 90) | oklch(0.10 0.01 250) | Body text on base | 4.5:1 |
| 2 | oklch(0.55 0.01 90) | oklch(0.10 0.01 250) | Muted text on base | 4.5:1 |
| 3 | oklch(0.55 0.01 90) | oklch(0.14 0.01 250) | Muted text on surface | 4.5:1 |
| 4 | oklch(0.92 0.01 90) | oklch(0.10 0.01 250) | Headings on base | 3:1 (large text) |
| 5 | oklch(0.85 0.24 145) | oklch(0.10 0.01 250) | Accent green on base | 4.5:1 |
| 6 | oklch(0.85 0.24 145) | oklch(0.14 0.01 250) | Accent green on surface | 4.5:1 |
| 7 | oklch(0.10 0.01 250) | oklch(0.85 0.24 145) | Dark text on green CTA | 4.5:1 |
| 8 | oklch(0.55 0.01 90) | oklch(0.18 0.01 250) | Muted text on elevated | 4.5:1 |

<task type="auto">
  <name>Task 2: Fix any failing contrast pairs by adjusting oklch token lightness</name>
  <files>src/styles/global.css</files>
  <action>
Based on the audit results from Task 1, update the failing tokens in `src/styles/global.css` @theme block.

**Expected failure:** `--color-text-muted: oklch(0.55 0.01 90)` — research predicts this pair is close to or below 4.5:1 against `oklch(0.10 0.01 250)`.

**Fix pattern:** Increase the L (lightness) value of the failing token until the pair achieves the required ratio. For muted text, try `oklch(0.62 0.01 90)` first. Re-run the audit script to verify.

**Rules:**
- Only change tokens that fail. Do NOT adjust passing tokens.
- Preserve the hue (H) and chroma (C) — only adjust lightness (L).
- Keep the visual hierarchy: muted text must remain visibly lighter than background but darker than body text. If body text is L=0.85 and muted goes to L=0.65, that hierarchy is maintained.
- After adjusting, re-run the audit script to confirm ALL pairs now pass.
- The stamp/accent-red is decorative — if it fails 3:1, increase its lightness slightly (e.g., `oklch(0.50 0.22 25)`).

**Leaflet popup hardcoded oklch values:** If the muted text token changes, also update the hardcoded `oklch(0.55 0.01 90)` values in the Leaflet CSS section of global.css (`.leaflet-popup.dark-popup .leaflet-popup-close-button`, `.leaflet-control-attribution`, `.leaflet-control-attribution a`, and `.pswp` `--pswp-icon-color-secondary`). These use raw oklch values instead of CSS variables — they must stay in sync with the token.

Delete the temporary audit script after verification.

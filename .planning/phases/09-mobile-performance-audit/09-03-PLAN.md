  <files>src/pages/index.astro, src/components/PhotoGallery.astro</files>
  <action>
Use Playwright (via MCP browser tools) to perform an automated layout audit at 375px width:

1. Start the dev server: `npm run dev` (ensure data is generated)
2. Navigate to `http://localhost:4321` with viewport set to 375x812 (iPhone SE dimensions)
3. Take a screenshot of the full page at 375px width
4. Check for horizontal overflow: execute JS in the page to detect if `document.documentElement.scrollWidth > document.documentElement.clientWidth`
5. Scroll through each section and verify:
   - Hero section: text is readable, CTA button is visible, countdown timer fits
   - Map section: map renders, GestureHandling message appears on touch
   - Elevation profile: chart is visible and not clipped
   - Sectors section: sector cards stack vertically, star ratings visible
   - Photo gallery: 2-column grid renders, thumbnails are tappable size (at least ~150px wide)
   - Event info: text is readable, GPX download button is full-width or at least tappable

6. For each section, check that no text is clipped or overflowing its container at 375px.

If any issues are found:
- Fix horizontal overflow by adding `overflow-x-hidden` if needed, or adjusting padding/grid
- Fix text overflow with word-break or truncation
- Ensure the photo gallery grid uses `grid-cols-2` (not higher) at 375px — verify the existing `grid-cols-2 md:grid-cols-3 lg:grid-cols-4` responsive classes work correctly

Document all findings. Fix any issues directly in the source files.

  <name>Task 2: Create CountdownTimer.astro component</name>
  <files>src/components/CountdownTimer.astro</files>
  <action>
Create `src/components/CountdownTimer.astro` with the following structure:

**Frontmatter:**
```astro
---
// Target: June 7, 2026 at 9:00 AM CDT (UTC-5)
// CDT = Central Daylight Time (Marquette, MI is in Eastern but the event time is placeholder)
// Using America/New_York EDT offset for Marquette: UTC-4
const EVENT_DATE_ISO = '2026-06-07T09:00:00-04:00';
---
```

**Template:**
- A container `<div>` with `id="countdown"` and `data-target={EVENT_DATE_ISO}`
- Use classes: `flex items-center justify-center gap-6 text-accent-white font-mono`
- Three time unit blocks, each containing:
  - A `<span>` for the number value with classes `text-4xl md:text-6xl font-bold tabular-nums`
    and a unique id (`cd-days`, `cd-hours`, `cd-minutes`), initial text content `--`
  - A `<span>` for the label with classes `text-text-muted text-xs uppercase tracking-widest ml-1`
    containing "d", "h", "m" respectively
- Below the countdown, a `<p>` with `id="cd-done"` and `hidden` attribute, classes
  `text-accent-green text-lg uppercase tracking-widest text-center`, text:
  `"The day is here. Get out there."`

**Processed script (NOT is:inline, NOT define:vars):**
```html
<script>
  const container = document.getElementById('countdown') as HTMLElement | null;
  if (container) {
    const target = new Date(container.dataset.target!).getTime();
    const days = document.getElementById('cd-days')!;
    const hours = document.getElementById('cd-hours')!;
    const minutes = document.getElementById('cd-minutes')!;
    const done = document.getElementById('cd-done') as HTMLElement;

    const tick = () => {
      const diff = target - Date.now();
      if (diff <= 0) {
        container.hidden = true;
        done.hidden = false;
        return;
      }
      days.textContent = String(Math.floor(diff / 86400000));
      hours.textContent = String(Math.floor((diff % 86400000) / 3600000));
      minutes.textContent = String(Math.floor((diff % 3600000) / 60000));
    };

    tick(); // Immediate call — no flash of '--' values
    setInterval(tick, 1000);
  }
</script>
```

IMPORTANT: Use a processed `<script>` tag (no `is:inline` attribute). This gets bundled by Astro.
IMPORTANT: Do NOT use `define:vars` — it forces `is:inline` and kills bundling (research pitfall 1).
IMPORTANT: Call `tick()` immediately before `setInterval` to prevent the '--' flash (research pitfall 4).
IMPORTANT: Handle post-event state: when diff <= 0, hide countdown and show done message (research pitfall 5).
IMPORTANT: Use `tabular-nums` class on number spans to prevent layout shift as digits change width.

Note on timezone: Marquette, MI is in the Eastern time zone. EDT (summer) is UTC-4. The
`-04:00` offset is correct for June. If the event director later confirms a different start
time, only the `EVENT_DATE_ISO` constant needs updating.

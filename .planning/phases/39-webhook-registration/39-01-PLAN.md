<task type="auto">
  <name>Task 3: Register webhook, verify subscription, simulate deauth deletion</name>
  <files>No files modified (operational curl commands only)</files>
  <action>
This task has three sequential parts. Execute them in order.

**Part A: Check for existing subscription**

Run the GET check to see if a subscription already exists:
```
curl -G https://www.strava.com/api/v3/push_subscriptions \
  -d client_id=CLIENT_ID \
  -d client_secret=CLIENT_SECRET
```
(Replace CLIENT_ID and CLIENT_SECRET with values from Task 2.)

- If response is `[]` (empty array): No subscription exists. Proceed to Part B.
- If response contains an object with `callback_url` matching `https://mkultragravel.netlify.app/.netlify/functions/strava-webhook`: Subscription already exists and is correct. Record the `id` field, skip Part B, proceed to Part C.
- If response contains an object with a DIFFERENT `callback_url`: Delete it first with:
  ```
  curl -X DELETE "https://www.strava.com/api/v3/push_subscriptions/SUBSCRIPTION_ID?client_id=CLIENT_ID&client_secret=CLIENT_SECRET"
  ```
  Then proceed to Part B.

**Part B: Register the subscription**

```
curl -X POST https://www.strava.com/api/v3/push_subscriptions \
  -F client_id=CLIENT_ID \
  -F client_secret=CLIENT_SECRET \
  -F callback_url=https://mkultragravel.netlify.app/.netlify/functions/strava-webhook \
  -F verify_token=dfb4e6536c623010dc78e73202a19773
```

Expected success: `{"id": SUBSCRIPTION_ID}` — record this ID.

This curl triggers Strava to immediately send a GET to the callback URL with a challenge. The function must respond correctly (verified in Task 1). If registration fails with "callback not verified," the function is not responding — revisit Task 1.

If registration fails with "already has subscription," go back to Part A and handle the existing subscription.

**Part B2: Confirm subscription is active**

Run the GET check again to verify:
```
curl -G https://www.strava.com/api/v3/push_subscriptions \
  -d client_id=CLIENT_ID \
  -d client_secret=CLIENT_SECRET
```

Expected: Array with one object containing the subscription ID, the correct callback_url, and created_at/updated_at timestamps. This confirms HOOK-01 and HOOK-02.

**Part C: Simulate deauthorization event**

First, confirm the target athlete file exists in the GitHub repo:
```
curl -s -o /dev/null -w "%{http_code}" \
  -H "Authorization: Bearer GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  "https://api.github.com/repos/Sheppardjm/mkUltraGravel/contents/public/data/results/athletes/2262684.json"
```
(Note: If GITHUB_TOKEN is not available locally, check the file exists via `ls public/data/results/athletes/2262684.json` in the local repo instead, or via `curl https://raw.githubusercontent.com/Sheppardjm/mkUltraGravel/main/public/data/results/athletes/2262684.json` which works for public repos.)

If the file does NOT exist (404 or not found locally): The deauth test cannot run meaningfully. Report that the athlete file must be re-created first (developer must re-submit via OAuth flow). STOP.

If the file exists: Execute the simulated deauthorization POST:
```
curl -v -X POST https://mkultragravel.netlify.app/.netlify/functions/strava-webhook \
  -H 'Content-Type: application/json' \
  -d '{
    "aspect_type": "delete",
    "event_time": 1711900800,
    "object_id": 2262684,
    "object_type": "athlete",
    "owner_id": 2262684,
    "subscription_id": SUBSCRIPTION_ID,
    "updates": { "authorized": "false" }
  }'
```

CRITICAL: `"authorized": "false"` must be a STRING with quotes, not a JSON boolean. The function uses strict equality `=== 'false'`.

Replace SUBSCRIPTION_ID with the ID from Part A or Part B.

Expected response: HTTP 200, body: `EVENT_RECEIVED`

**Part C2: Verify deletion occurred**

Wait 10-15 seconds for the GitHub API call to complete, then verify:

1. Check GitHub repo for the file deletion:
```
curl -s -o /dev/null -w "%{http_code}" \
  "https://raw.githubusercontent.com/Sheppardjm/mkUltraGravel/main/public/data/results/athletes/2262684.json"
```
Expected: 404 (file deleted).

If still 200: Wait another 15 seconds and retry. GitHub CDN caching may delay. Also check the GitHub repo commits page for a commit with message "deauth: delete athlete 2262684 data per TOS 5.4".

2. Check for Netlify rebuild trigger:
The function fires a POST to the NETLIFY_BUILD_HOOK after deletion. Check the Netlify Deploys page for a new build triggered after the deletion commit. (This is informational — the rebuild may take 1-2 minutes to appear.)

Record all results: subscription ID, registration response, deauth curl response, file deletion confirmation, rebuild trigger status.

**Post-test note for developer:** After confirming deletion works, you should re-submit your activity via the OAuth flow (`/submit` page) to restore your leaderboard entry before Phase 40 (app review screenshots need a populated leaderboard).

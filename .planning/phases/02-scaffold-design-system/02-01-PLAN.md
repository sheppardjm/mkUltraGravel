<task type="auto">
  <name>Task 2: Create astro.config.mjs and global CSS with cascade layers</name>
  <files>astro.config.mjs, src/styles/global.css, tsconfig.json</files>
  <action>
  1. Create `astro.config.mjs` at project root:
     ```javascript
     import { defineConfig } from "astro/config";
     import tailwindcss from "@tailwindcss/vite";

     export default defineConfig({
       vite: {
         plugins: [tailwindcss()],
       },
     });
     ```
     NOTE: Do NOT add the Fonts API config yet — that comes in Plan 02-02 after
     the font selection is confirmed. Keep this file minimal for now.

  2. Create `src/styles/global.css`:
     ```css
     /* Cascade layer order — declared FIRST, before any imports.
        Leaflet CSS (Phase 3) goes in the lowest-priority layer so
        Tailwind utilities always win. */
     @layer leaflet, base, components, utilities;

     /* Tailwind v4 — CSS-first config, no tailwind.config.js */
     @import "tailwindcss";

     /* Leaflet CSS will be imported here in Phase 3:
        @import "leaflet/dist/leaflet.css" layer(leaflet); */

     /* Design tokens — populated in Plan 02-02 */
     @theme {
       /* Colors, fonts, spacing defined in next plan */
     }
     ```

  3. Create `tsconfig.json` at project root (Astro expects this):
     ```json
     {
       "extends": "astro/tsconfigs/strict"
     }
     ```
     Astro uses this for .astro file type checking even in a JS project.

  4. Create a minimal `src/pages/index.astro` so `astro dev` has something to serve:
     ```astro
     ---
     import "../styles/global.css";
     ---
     <html lang="en">
       <head>
         <meta charset="utf-8" />
         <meta name="viewport" content="width=device-width, initial-scale=1" />
         <title>MK Ultra Gravel</title>
       </head>
       <body>
         <h1>MK Ultra Gravel</h1>
         <p>Scaffold working. Design system incoming.</p>
       </body>
     </html>
     ```
     This is a throwaway page — Plan 02-02 will replace it with BaseLayout + proper index.
     Its only purpose is to verify the Astro + Tailwind pipeline works end-to-end.

  5. Ensure the `src/` directory structure exists:
     ```
     src/
     ├── layouts/     (empty for now — Plan 02-02)
     ├── components/  (empty for now)
     ├── pages/
     │   └── index.astro
     └── styles/
         └── global.css
     ```
  </action>

Read route-data.json at build time using `fs.readFileSync` in the Astro frontmatter of index.astro, then display distance and elevation gain in the #route section heading.

**Part A: Add fs imports and data reading to frontmatter**

At the TOP of the frontmatter block (line 2, before component imports), add:
```astro
import { readFileSync } from 'node:fs';
import { resolve } from 'node:path';

const routeDataPath = resolve('./public/data/route-data.json');
const routeDataJson = JSON.parse(readFileSync(routeDataPath, 'utf8'));
const routeMeta = routeDataJson.meta;
```

Use `node:fs` and `node:path` (Node.js built-in protocol prefix) — this is the modern convention and works in Astro SSG.

**Part B: Add route stats subtitle to #route section**

In the `<section id="route">` template, immediately AFTER the `<h2>The Route</h2>` heading and BEFORE the `<RouteMap />` component, add a subtitle paragraph:

```html
<p class="text-text-muted text-sm mb-8">
  {Math.round(routeMeta.totalMi)} miles &mdash; {routeMeta.elevationGainFt.toLocaleString()} ft elevation gain
</p>
```

This replaces the bare `<h2>The Route</h2>` with a heading + subtitle pattern. The `Math.round()` call rounds 98.23 to 98 (will auto-update to 100 when the new GPX is processed). The `.toLocaleString()` adds comma formatting to the elevation number (e.g., "3,189").

**Important notes:**
- Display the LIVE JSON value, do NOT hardcode "100 miles"
- The hero still says "100 miles" — that's a separate hardcoded string, leave it alone
- The route stats will auto-update when the GPX pipeline regenerates route-data.json

Switch masonry gallery from sequential column fill to balanced column fill so the last column
is not dramatically shorter than its neighbors.

activating column-fill at all. Two CSS property changes fix both issues.

  <files>src/components/PhotoGallery.astro</files>
  <action>
In the `<style>` section of `PhotoGallery.astro`, make two property changes at every breakpoint
where `.masonry-gallery` is styled:

1. **Base rule** (~line 46-54):
   - Change `max-height: 75vh` to `height: 75vh`
   - Change `column-fill: auto` to `column-fill: balance`
   - Remove the comment about column-fill (it's no longer relevant)

2. **480px breakpoint** (~line 56-60):
   - Change `max-height: 80vh` to `height: 80vh`

3. **768px breakpoint** (~line 62-69):
   - Change `max-height: 85vh` to `height: 85vh`

4. **1280px breakpoint** (~line 71-76):
   - Change `max-height: 90vh` to `height: 90vh`

**Why `height` not `max-height`:** Chrome only activates `column-fill` (both `auto` and
`balance`) when the container has an explicitly defined block-dimension size. `max-height`
does not qualify — Chrome ignores `column-fill` entirely with `max-height`. Since 73 photos
always exceed any breakpoint height, a fixed `height` is safe and functionally equivalent.

**Do NOT change:**
- `columns` shorthand values (column count / min-width)
- `column-gap` values
- `overflow-x: auto` or `overflow-y: hidden`
- Any `.gallery-item`, `.gallery-img`, or `.gallery-mile` styles
- Any PhotoSwipe selectors (`#photo-gallery`, `.gallery-item`)
- The `<script>` section

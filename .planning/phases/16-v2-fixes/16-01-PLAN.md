  <files>src/styles/global.css</files>
  <action>
Make these four changes to src/styles/global.css:

**Fix 1 — PhotoSwipe lightbox layer ordering (Gap 2):**
- Line 4: Remove 'photoswipe' from the @layer declaration. Change:
  `@layer leaflet, photoswipe, base, components, utilities;`
  to:
  `@layer leaflet, base, components, utilities;`
- Line 13: Move PhotoSwipe import from layer(photoswipe) to layer(components). Change:
  `@import "photoswipe/style.css" layer(photoswipe);`
  to:
  `@import "photoswipe/style.css" layer(components);`
- Lines 210-212: Remove the redundant `.pswp__img { max-width: none; }` override entirely (the block that was compensating for the layer ordering issue). The PhotoSwipe CSS now lives in the components layer alongside the dark-theme overrides, so Tailwind Preflight in the base layer no longer overrides it.

**Fix 2 — Card hover shadow (Gap 6):**
Replace the entire `.card-hover` block (lines ~227-250) with a direct box-shadow approach that is not clipped by overflow-hidden and not hidden behind opaque backgrounds:

```css
/* Card hover -- brutalist hard-shift shadow (direct box-shadow, not clipped by overflow-hidden) */
.card-hover {
  box-shadow: 4px 4px 0 0 transparent;
  transition: box-shadow 0ms step-start;
}
.card-hover:hover {
  box-shadow: 4px 4px 0 0 var(--color-accent-green);
}
@media (prefers-reduced-motion: reduce) {
  .card-hover {
    transition: none;
  }
  .card-hover:hover {
    box-shadow: 4px 4px 0 0 transparent;
  }
}
```

Remove the entire ::after pseudo-element approach (`position: relative`, `::after`, `:hover::after`, and its prefers-reduced-motion rules). The ::after trick failed because overflow-hidden on card containers clips the pseudo-element, and z-index:-1 puts the shadow behind the opaque card background.

**Fix 3 — Scroll reveal keyframes (Gap 7):**
Move `@keyframes reveal` OUT of the `@theme {}` block. Tailwind v4's @theme only processes design tokens (custom properties) — it does NOT emit @keyframes rules.

- Keep `--animate-reveal: reveal 0.35s ease-out both;` INSIDE @theme (it is a design token).
- Remove the `@keyframes reveal { ... }` block from inside @theme (lines 36-45).
- Add it as a top-level block AFTER the @theme closing brace (before `@layer base`):

```css
@keyframes reveal {
  from {
    opacity: 0;
    transform: translateY(12px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

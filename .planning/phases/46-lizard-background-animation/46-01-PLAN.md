  <action>
Create `src/components/LizardBackground.astro` as a fixed-position overlay with a repeating lizard SVG tile encoded as a CSS data URI background-image.

**Component structure:**
- A single `<div class="lizard-bg" aria-hidden="true"></div>`
- The `.lizard-bg` class is defined in the component's scoped `<style>` block for the base properties (position, inset, pointer-events, z-index, opacity, background-image, background-repeat, background-size, will-change)
- The `@keyframes lizard-drift` and its `@media (prefers-reduced-motion: no-preference)` gate go in `src/styles/global.css` — matching the pattern of `@keyframes escher-drift` which is also in global.css outside any layer

**SVG tile construction:**
- Extract the 4 lizard `<g>` elements (with their transforms) from `src/components/EscherLizards.astro`'s `<pattern id="escher-lizard-tile">` (lines 14-28)
- Wrap them in a standalone `<svg xmlns='http://www.w3.org/2000/svg' width='200' height='200'>` — NO `<defs>`, NO `<pattern>`, NO `<rect fill="url(#...)">`. Place the 4 `<g>` elements directly as children of the `<svg>`. This avoids the `url(#id)` internal reference encoding problem entirely.
- Encode for data URI: replace `"` with `'`, `#` with `%23`, `<` with `%3C`, `>` with `%3E`
- Set as `background-image: url("data:image/svg+xml,...")`

**CSS properties for `.lizard-bg`:**
- `position: fixed; inset: 0; pointer-events: none;`
- `z-index: 9997` (below escher-overlay at 9998, below grain-overlay at 9999)
- `opacity: 0.04` (starting value — below grain 0.06 and escher 0.05, calibrated for three-layer stack)
- `background-repeat: repeat; background-size: 200px 200px;`
- `will-change: transform;`

**Keyframes in global.css** (add after the existing `@keyframes escher-drift` block, around line 272):
```css
@keyframes lizard-drift {
  0%   { transform: translate(0, 0); }
  100% { transform: translate(200px, 200px); }
}

@media (prefers-reduced-motion: no-preference) {
  .lizard-bg {
    animation: lizard-drift 80s linear infinite;
  }
}
```
Note: translate distance (200px, 200px) exactly matches background-size (200px 200px) for seamless loop. Duration 80s is slower than escher-drift (50s) to differentiate drift speeds and keep the lizard layer more subliminal.

**BaseLayout.astro integration:**
- Import `LizardBackground` from `../components/LizardBackground.astro`
- Add `<LizardBackground />` in the `<body>` after the escher-overlay div (around line 37), before `<slot />`

**What to avoid and why:**
- Do NOT animate `background-position` — it triggers paint on every frame, not compositor-safe, will hurt Lighthouse and cause stutter on mobile
- Do NOT use `<pattern>` + `url(#id)` inside the data URI SVG — the `#` in internal references creates encoding headaches; inline the paths directly
- Do NOT use `mix-blend-mode` — on a near-black background at 4% opacity, blend modes add complexity without benefit
- Do NOT use JavaScript for animation — any synchronous JS = TBT risk; CSS-only guarantees 0ms TBT
- Do NOT use base64 encoding — 33% size overhead vs percent-encoding; the project uses percent-encoding for both existing overlays
- Do NOT place the keyframes inside the component's scoped style — Astro scoped styles hash class names, but `@keyframes` need to be available globally since the animation property references the keyframe name; follow the escher-drift pattern of global.css

Create `scripts/match-photos.js` that:

1. Reads `public/data/route-data.json`. If not found, print error and exit(1).

2. Imports `photoManifest` from `./photo-manifest.js` (same scripts/ directory).

3. Implements `findPointAtMile(routeData, targetMile)` -- same function as in
   resolve-annotations.js. (Duplication is fine for two small scripts; no need for a
   shared module. Keep each script self-contained and runnable independently.)

4. Optionally attempts EXIF GPS extraction using exifr as a forward-compatible pattern:
   ```javascript
   const exifr = require('exifr');
   const path = require('path');

   async function getPhotoPosition(filename, mileFallback, routeData) {
     const imagePath = path.join('images', filename);
     // Attempt EXIF GPS (will return undefined for all current photos)
     const gps = await exifr.gps(imagePath).catch(() => undefined);
     if (gps && gps.latitude && gps.longitude) {
       return { lat: gps.latitude, lon: gps.longitude, source: 'exif' };
     }
     // Manual fallback via mile marker
     const pt = findPointAtMile(routeData, mileFallback);
     return { lat: pt.lat, lon: pt.lon, source: 'manual' };
   }
   ```

5. Processes all 33 manifest entries:
   ```javascript
   const photos = [];
   for (const entry of photoManifest) {
     const pos = await getPhotoPosition(entry.filename, entry.mi, routeData);
     photos.push({
       filename: entry.filename,
       lat: pos.lat,
       lon: pos.lon,
       mi: entry.mi,
       source: pos.source
     });
   }
   ```

6. Sorts output by mi ascending (should already be sorted from manifest, but enforce it).

7. Validates before writing:
   - Exactly 33 entries
   - All filenames exist in images/ directory (fs.existsSync check)
   - No duplicate filenames
   - Log any issues as warnings

8. Writes `public/data/photos.json` with `JSON.stringify(photos, null, 2)`.

9. Logs summary:
   - Total photos processed
   - Count by source (expect: 0 exif, 33 manual)
   - Mile range covered (first mi to last mi)

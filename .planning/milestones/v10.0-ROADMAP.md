# Milestone v10.0: Strava Decoupling

**Status:** ✅ SHIPPED 2026-04-06
**Phases:** 48-49
**Total Plans:** 3

## Overview

Remove Strava OAuth submission, scoring engine, and results leaderboards from MK Ultra. Replace /results with CTA to Iron & Pine Omnium (ironpineomnium.com). MK Ultra becomes a pure static event site with no Netlify Functions dependency.

## Phases

### Phase 48: Strava Infrastructure Removal

**Goal**: All Strava OAuth, scoring, submission, and results code is removed from the codebase
**Depends on**: Nothing (first phase of v10.0)
**Plans**: 2 plans

Plans:
- [x] 48-01: Delete Strava-dependent pages, ScoringExplainer, clean index.astro and KomSegments.astro
- [x] 48-02: Delete scoring engine, Netlify Functions, results data, clean package.json and netlify.toml

**Details:**

**Requirements:** REM-01, REM-02, REM-03, REM-04, REM-05, REM-06, CLN-01

**Success Criteria:**

**Completed:** 2026-04-06

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Delete page files before their backend dependencies (results.astro before scoring.js) to prevent cascading build failures
- Preserve Strava segment links on KOM cards — only remove time data fetched from Strava API
- Remove "test" script from package.json alongside "validate" — vitest exits non-zero with no test files
- Retain [[redirects]] in netlify.toml — /api/* redirect is a harmless no-op
- Retain [build.environment] block — SECRETS_SCAN_OMIT_PATHS still protects .planning/
- Results page defers to ironpineomnium.com instead of hosting local leaderboard
- isActive() reduced to single-line exact match — no special-casing after /submit removal

**Issues Resolved:**
- Strava TOS compliance risk eliminated by moving all Strava-dependent features to Iron & Pine Omnium
- REVIEW-03 (Strava app approval) no longer gates MK Ultra — external dependency removed
- Node.js version mismatch (v20 shell default vs Astro >=22 requirement) — worked around with explicit paths

**Issues Deferred:**
- Card cover photo aspect ratios on non-accent cards (noted during Phase 44 checkpoint)
- Netlify env var cleanup (manual step in Netlify dashboard, not code)

**Technical Debt Incurred:**
- None — this was a pure subtraction milestone

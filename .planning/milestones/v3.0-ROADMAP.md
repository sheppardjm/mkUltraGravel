## Milestone Summary

**Key Decisions:**
- starColors not extracted to shared module — three-file duplication acceptable for 5-line constant
- Hex fills in favicon.svg instead of oklch() — SVG oklch browser support inconsistent
- L.marker + L.divIcon for bike crosshair (not L.circleMarker) — enables arbitrary SVG
- Four standalone `<rect>` elements in SVG tile instead of `<use>` refs — data URIs can't resolve fragment identifiers
- KOM annotations intentionally omit _baseColor for sector event isolation

**Issues Resolved:**
- 33/53 photo mile markers corrected from wildly inaccurate estimates
- Gray sector colors replaced with perceptually distinct yellow-to-red spectrum
- Elevation chart now shows both sector and KOM segment context

**Issues Deferred:**
- Down Jeep KOM (mi 83.55-84.15) uses nearest fallback photo at mi 80.2 — no in-range photo available
- Billie Helmer KOM (mi 21.9-22.59) uses nearest fallback at mi 21.1 — position shift from corrections
- onHover performance on mid-range Android unverified
- Billie Helmer sector card photo replacement pending photo from route owner (VIS-16)
- Bike icon bearing rotation based on route direction (VIS-17)

**Technical Debt Incurred:**
- starColors constant duplicated in 3 files (RouteMap.astro, ElevationProfile.astro, GravelSectors.astro) — extraction deferred
- Node.js version mismatch in local dev environment (v20 default vs ≥22 required) — workaround via PATH override

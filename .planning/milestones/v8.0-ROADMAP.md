# Milestone v8.0: Visual Polish + Content

**Status:** ✅ SHIPPED 2026-04-01
**Phases:** 41-46
**Total Plans:** 6

## Overview

Elevate the site's visual texture and photo presentation with six additive features — updated route data, 16 new photos, horizontal masonry gallery, tone image integration, topographic meatball dividers, and animated lizard background — without touching the existing brutalist identity or regressing Lighthouse mobile 96 / TBT 0ms / CLS 0.054 baselines.

## Phases

### Phase 41: GPX Route Update + Pipeline Validation

**Goal**: The route data source is replaced and all downstream artifacts verified correct — hero stats, elevation profile, map polyline, sector/KOM coordinates, and photo positions all derive from the new GPX file.
**Depends on**: Phase 40 (v7.0 complete)
**Plans**: 1 plan

Plans:
- [x] 41-01: Replace GPX source with MKULTRA.gpx, regenerate pipeline, verify artifacts, remove old GPX

**Details:**
Single-line change to `scripts/parse-gpx.js` switches route source from MK_Ultra.gpx to MKULTRA.gpx. Pipeline produced 2581 trackpoints, 100.62 miles, 3365 ft gain. All 9 annotation mile markers resolved without clamping. All 55 photo positions within bounds. Old GPX removed via `git rm`.

### Phase 42: Photo Pipeline Expansion

**Goal**: All route photos are processed through the pipeline with correct mile markers, thumbnails, card crops, and cover photo assignments — the gallery and card components have full data before any layout work begins.
**Depends on**: Phase 41 (pipeline clean on new GPX)
**Plans**: 1 plan

Plans:
- [x] 42-01: Add 16 photos to manifest and regenerate pipeline artifacts (3 excluded by owner)

**Details:**
Expanded photo manifest from 55 to 71 entries with 16 owner-assigned mile markers. Route coverage extended from mi 19.6–83.8 to mi 13.8–95.4. All 9 cover photo assignments unchanged after expansion. 3 candidate photos excluded by route owner as not route-relevant.

### Phase 43: Horizontal Masonry Gallery

**Goal**: The photo gallery displays all photos in their natural aspect ratios as a masonry grid — no more fixed-crop vertical grid that damages landscape photos.
**Depends on**: Phase 42 (71 photos with correct dimensions in photos.json)
**Plans**: 1 plan

Plans:
- [x] 43-01: Refactor PhotoGallery.astro to CSS columns masonry grid with scroll-snap

**Details:**
CSS columns masonry layout with `column-count` + `break-inside-avoid-column`. Max-height constraint with overflow-x scroll. Aspect-ratio placeholders prevent CLS during lazy load. PhotoSwipe lightbox preserved. Pivoted from horizontal flex strip to masonry grid at user checkpoint.

### Phase 44: Tone Image Integration

**Goal**: Tone images appear as full-width interstitial dividers between major sections and as subtle card accents inside 2-3 sector or KOM cards.
**Depends on**: Phase 41 (pipeline clean; tone image pipeline script uses same prebuild chain)
**Plans**: 1 plan

Plans:
- [x] 44-01: SVG lizard tessellation on #sectors, card accents on 2 sector + 1 KOM card

**Details:**
EscherLizards.astro SVG `<pattern>` tessellation on #sectors section at opacity 0.12 with mix-blend-mode: lighten. Tone accents on first 2 sector cards (Sandstrom Rd, Akkala Rd) and first KOM card (Billie Helmer) with isolation: isolate containment. Pivoted from raster WebP to SVG at checkpoint for resolution independence.

### Phase 45: Topographic Meatball Dividers

**Goal**: Hollow topographic SVG meatballs appear between 2 or more sections, drawing in as the user scrolls.
**Depends on**: Phase 44 (section containers settled with tone image stacking contexts)
**Plans**: 1 plan

Plans:
- [x] 45-01: Build TopoDivider.astro component and place between sections

**Details:**
Canvas + SVG filter metaball approach: 6 orbiting blobs through feGaussianBlur + feComponentTransfer (discrete alpha bands) produce concentric topographic contour rings. IntersectionObserver starts/stops requestAnimationFrame loop. Placed between #route/CTA and #sectors/#photos. Pivoted from stroke-dashoffset circles to canvas metaballs at user checkpoint.

### Phase 46: Lizard Background Animation

**Goal**: A subtly animated lizard tessellation layer sits behind all site content at z-index 9997, extending the existing Escher overlay language with a second motif — and all Core Web Vitals verified green with the full v8.0 texture stack.
**Depends on**: Phase 45 (full v8.0 visual stack assembled for opacity calibration)
**Plans**: 1 plan

Plans:
- [x] 46-01: Build LizardBackground.astro and run final performance audit

**Details:**
LizardBackground.astro with 4 lizard SVG paths percent-encoded as CSS data URI background-image. 80s diagonal drift via transform-only keyframes. Opacity 0.04 (below grain 0.06, escher 0.05). mask-image linear-gradient hides lizard from hero section. Lighthouse mobile 96, TBT 0ms, CLS 0.073. Three-layer texture stack complete: grain (9999) > escher (9998) > lizard (9997).

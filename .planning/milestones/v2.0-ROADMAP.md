# Milestone v2.0: Interactivity + Polish

**Status:** SHIPPED 2026-03-28
**Phases:** 11-16
**Total Plans:** 15

## Overview

A 6-phase milestone that elevated the site from informational to interactive. Data corrections
fixed segment positions and photo mile markers before any UI work. The photo pipeline automated
card crop generation and thumbnail upgrades. Bidirectional map-elevation sync made the route
explorable. Content additions (MK Ultra explainer, route stats, activated URLs) filled gaps.
Brutalist animations added personality without regressing performance. UAT fixes closed CSS
regressions and visual hierarchy gaps found during verification.

## Milestone Summary

**Key Decisions:**

- Data corrections must precede all card/photo/map work — building UI on bad data guarantees rework
- Strava live leaderboard permanently dropped — TOS prohibits displaying other athletes' data; endpoint blocked since June 2020
- Map-elevation interactivity isolated to measure Lighthouse TBT delta cleanly before animations added
- assign-card-photos.js uses Haversine proximity matching — no new npm dependency
- coverPhoto stored as filename only in annotations.json — components construct path at render time
- All animations CSS-only (transform/opacity) — protects TBT 0ms; no GSAP or motion library
- window CustomEvent bus (not shared module) keeps ElevationProfile/RouteMap decoupled
- chart.update('none') mandatory on all annotation mutations to maintain TBT 0ms
- Direct box-shadow on .card-hover (not ::after) — overflow-hidden clips pseudo-elements
- Two-div structural pattern for card-hover + overflow-hidden (CSS Overflow Module Level 3)
- PhotoSwipe CSS in layer(components) not layer(photoswipe) — Tailwind Preflight override fix
- @keyframes must be top-level in Tailwind v4 — @theme silently drops them

**Issues Resolved:**

- Segment mile markers corrected (Down Jeep, Silver Creek, others)
- Photo mile-marker positions corrected for 53 photos
- Laughing Whitefish River removed from restock points
- PhotoSwipe lightbox layer ordering (CSS cascade layer fix)
- Card hover shadow clipped by overflow-hidden (structural two-div fix)
- Scroll-reveal keyframes dropped by @theme nesting (moved to top-level)
- Elevation crosshair broken (getRelativePosition import from chart.js/helpers, not chart.js/auto)
- Leaving Chatham KOM no longer uses nearest-photo fallback (dedicated photo at mi 37.8)

**Issues Deferred:**

- Down Jeep KOM (mi 83.55-84.15) still uses nearest fallback at mi 80.2 — no in-range photo available
- onHover performance on mid-range Android unverified — must test with real device

**Technical Debt Incurred:**

- Hardcoded raw oklch values in Leaflet/PhotoSwipe CSS must stay in sync with @theme tokens (carried from v1.0)
- Scripts intentionally self-contained (findPointAtMile duplicated across scripts) (carried from v1.0)

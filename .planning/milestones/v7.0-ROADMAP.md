# Milestone v7.0: Strava Go-Live

**Status:** ✅ SHIPPED 2026-03-31
**Phases:** 36-40
**Total Plans:** 5

## Overview

Get the Strava submission pipeline working end-to-end with real data — a real athlete can OAuth in, submit an activity, and appear on the leaderboard.

This is an operational milestone, not a build milestone. The pipeline is fully implemented from v5.0. The work is configuration, verification, and reactive bug fixes found during real-data testing.

## Milestone Summary

**Key Decisions:**
- Netlify Functions v1 (exports.handler) kept as-is — v2 env var bug too recent for risk-free migration
- STRAVA_REDIRECT_URI must be full URL, not relative path or /api/ rewrite alias
- STRAVA_VERIFY_TOKEN = random 32-char hex for webhook verification
- Added SECRETS_SCAN_OMIT_PATHS/KEYS to netlify.toml — Netlify secrets scanning blocked builds
- Static Astro pages receiving server redirect data must parse query params client-side, not in frontmatter
- Strava branding uses exact #FC5200 hex, never oklch approximation

**Issues Resolved:**
- Netlify secrets scanning blocking all builds (fixed with SECRETS_SCAN_OMIT_PATHS in netlify.toml)
- submit-confirm.astro query param parsing (moved from build-time frontmatter to client-side JS)
- STRAVA_REDIRECT_URI format (changed from relative to full URL)
- oklch(0.72 0.19 55) color approximation (replaced with exact #FC5200 across 3 files)

**Issues Deferred:**
- REVIEW-03: Strava app approval externally gated (submitted 2026-03-31, expected ~April 7-14). If not approved by ~May 28, escalate to developers@strava.com.

**Technical Debt Incurred:**
- None — operational milestone with minimal code changes

# Milestone v1.0: MK Ultra Gravel

**Status:** SHIPPED 2026-03-27
**Phases:** 1-10
**Total Plans:** 30

## Overview

A 10-phase build that starts with the load-bearing data pipeline and ends with a deployed,
audited site ready to convert gravel cyclists into registered participants for June 7, 2026.
The data pipeline feeds everything — map, annotations, photo markers, gallery, static route
cards — so it executes first. The map is the centrepiece feature and gets the second major
slot. Static content sections follow independently. Mobile audit and deployment close the build.

## Milestone Summary

**Key Decisions:**

- Paris-Roubaix sector rating system for gravel difficulty communication
- Leaflet 1.9.4 with Carto Dark Matter tiles (no API key required)
- oklch color space for perceptually uniform dark palette tokens
- prebuild npm lifecycle hook for data generation pipeline
- Netlify over Cloudflare Pages for deployment (user preference)
- Custom domain deferred — shipping with netlify.app URL
- BikeReg URL placeholder pending event director confirmation

**Issues Resolved:**

- All 33 photos lack EXIF GPS — used manual mile-marker positioning
- gpxparser cumul array alignment (prepend 0, slice last)
- Tailwind v4 / Vite 7 peer dependency conflict (overrides.vite=^7)
- PhotoSwipe lightbox collapsed to 0px (Tailwind preflight conflict)
- Lighthouse CSS inlining backfired (44KB inline > render-blocking RTT savings)

**Issues Deferred:**

- BikeReg registration URL not confirmed — CTAs use PENDING placeholder
- Custom domain setup — can be added via Netlify dashboard anytime

**Technical Debt Incurred:**

- Hardcoded raw oklch values in Leaflet/PhotoSwipe CSS must stay in sync with @theme tokens
- Scripts intentionally self-contained (findPointAtMile duplicated across scripts)

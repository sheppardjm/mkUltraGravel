# Milestone v6.0: UI Polish + Dev Tools

**Status:** SHIPPED 2026-03-30
**Phases:** 33-35
**Total Plans:** 3

## Overview

Fix color inconsistencies across map/chart/cards, label gravel sectors on the elevation profile, and add a site navigation header for wayfinding between pages.

## Phases

### Phase 33: Color Consistency

**Goal**: All star-rating colors are sourced from a single shared module — map polylines, elevation bands, and sector cards show identical colors for every rating.

**Completed**: 2026-03-30

**Requirements**: CLR-01, CLR-02, CLR-03

Plans:

- [x] 33-01: Extract starColors to src/lib/starColors.ts and update all three consumers

**Details:**
Created `src/lib/starColors.ts` as single source of truth for the 5-color Paris-Roubaix palette. Replaced 3 duplicate inline object literals in GravelSectors.astro, RouteMap.astro, and ElevationProfile.astro with shared imports. Module works in both SSR (Astro frontmatter) and browser (script tags).

---

### Phase 34: Elevation Profile Sector Labels

**Goal**: Every gravel sector on the elevation profile is identified by name and star rating — a user can read the chart and know which sectors are ahead without cross-referencing the sector cards.

**Completed**: 2026-03-30

**Requirements**: ELEV-01, ELEV-02, ELEV-03, ELEV-04

Plans:

- [x] 34-01: Add sector label annotations to ElevationProfile.astro with stagger strategy

**Details:**
Added Chart.js annotation label sub-objects to all 6 gravel sector bands. Labels show sector name + Unicode star rating at bottom of chart. Adjacent labels staggered via yAdjust i%2 alternation. Down Jeep (0.60mi) gets stars-only content with rotation:-90. Label color uses starColor + 'cc' (80% opacity).

---

### Phase 35: Site Navigation

**Goal**: Every page on the site has a fixed navigation header — a user can reach Home, Results, and Submission from any page without using the browser back button, and can see at a glance which page they are on.

**Completed**: 2026-03-30

**Depends on**: Phase 32 (v5.0 complete; independent of Phases 33-34)

**Requirements**: NAV-01, NAV-02, NAV-03, NAV-04

Plans:

- [x] 35-01: Build SiteNav.astro component and integrate into BaseLayout.astro

**Details:**
Created SiteNav.astro with fixed positioning and build-time active link detection via Astro.url.pathname. Nav renders above grain (9999) and Escher (9998) overlays at z-index 10000. Removed ad-hoc "Back to" links from results, submit, and submit-confirm pages. BaseLayout.astro updated with pt-12 body padding for fixed nav clearance.

---

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- Decision: starColors extracted to src/lib/starColors.ts (Rationale: Single source of truth for color palette; follows src/lib/ pattern)
- Decision: Annotation label sub-object over CSS overlay strip (Rationale: y-axis alignment reliable; keeps logic co-located with annotation data)
- Decision: Astro.url.pathname for active link detection (Rationale: No client JS, no FOUC, fully static-safe)
- Decision: z-index 10000 for nav (Rationale: Must clear grain 9999 and Escher 9998 overlays)
- Decision: /submit-confirm grouped with /submit in isActive() (Rationale: Both are part of submission flow)

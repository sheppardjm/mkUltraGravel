# Milestone v4.0: Route Update + UX Overhaul

**Status:** ✅ SHIPPED 2026-03-30
**Phases:** 22-26
**Total Plans:** 7

## Overview

Update the route from 80mi to 100mi (cascading through the entire data pipeline), add two new photos, then deliver five UX improvements ranging from CSS quick wins to a PhotoSwipe-Leaflet integration -- all within the existing stack with zero new dependencies.

## Phases

### Phase 22: GPX Route Replacement

**Goal**: The site displays the correct 100-mile route with all sectors, KOMs, restocks, and photos resolved to accurate positions on the new track.
**Depends on**: Nothing (first v4.0 phase). BLOCKED until new GPX file arrives from Strava.
**Requirements**: ROUTE-04, ROUTE-05, ROUTE-06
**Plans**: 1 plan

Plans:
- [x] 22-01: Swap GPX source, regenerate pipeline, fix display rounding, verify route

**Details:**
- Swapped GPX source from `MK Ultra.gpx` (80mi) to `MK_Ultra.gpx` (100mi)
- Full pipeline re-run: route-data.json (100.71mi, 2779 trackpoints, 3595ft gain), annotations.json, photos.json
- Math.floor for distance display (100.71 → "100 miles"), Math.ceil for chart x-axis max
- All 6 sectors/3 KOMs confirmed within shared track geometry (below mi 84.15)
- Old GPX removed from git tracking

### Phase 23: New Photos

**Goal**: Two new photos (Down Jeep + Billie Helmer B&W) are fully integrated — visible on the map, in the gallery, and assigned to cards where applicable.
**Depends on**: Phase 22 (photos must be positioned against the 100mi route geometry)
**Requirements**: PHOTO-01, PHOTO-02
**Plans**: 1 plan

Plans:
- [x] 23-01: Add AVIF support, manifest entries, and run pipeline for Down Jeep + Billie Helmer photos

**Details:**

### Phase 24: CSS + Layout + Content

**Goal**: Four independent visual and content improvements ship together — larger touch-friendly zoom controls, equalized card sizes, Penrose triangle branding in the hero, and a Grinduro format explainer for first-time visitors.
**Depends on**: Nothing (parallel-safe, no data dependencies on Phase 22/23)
**Requirements**: MAP-10, LAYOUT-01, LAYOUT-02, CONT-06
**Plans**: 3 plans

Plans:
- [x] 24-01: Zoom control 44px touch targets (MAP-10) + card height equalization (LAYOUT-01)
- [x] 24-02: Penrose triangle hero SVG (LAYOUT-02) + Grinduro format explainer (CONT-06)
- [x] 24-03: Gap closure: card height h-[180px], reset button as Leaflet control, zoom enlarged to 52px

**Details:**
- Leaflet zoom controls enlarged from default to 52px with dark theme
- Card height equalized via h-[180px] fixed image height (decoupled from container width)
- Penrose triangle SVG with 20s rotation animation, gated on prefers-reduced-motion
- GrinduroExplainer.astro component describing timed sectors, KOM/QOM, untimed connectors
- Reset button moved to custom Leaflet control (L.Control.extend) per user feedback

### Phase 25: Map Reset

**Goal**: Users can return the map and elevation chart to their default state with a single click, clearing all highlights, popups, and zoom changes.
**Depends on**: Phase 22 (reset needs correct default bounds from new route data to verify)
**Requirements**: MAP-09
**Plans**: 1 plan

Plans:
- [x] 25-01: Reset button + map/elevation reset handlers via CustomEvent bus

**Details:**
- map:reset CustomEvent dispatched from reset button, handled by RouteMap + ElevationProfile
- map.fitBounds (not flyTo) guarantees pixel-identical framing to initial page load
- initialBounds captured before any user interaction
- Sector polyline styles, crosshair, popups, elevation band opacity all restored
- KOM annotations skipped by _baseColor guard (no state to reset)

### Phase 26: Photo Lightbox from Map

**Goal**: Clicking a photo marker on the map opens a full-screen PhotoSwipe lightbox instead of opening a new browser tab, with swipe navigation through all route photos.
**Depends on**: Phase 23 (photos.json must include all 55 photos with width/height data)
**Requirements**: MAP-11, MAP-12
**Plans**: 1 plan

Plans:
- [x] 26-01: Thumbnail photo markers + programmatic PhotoSwipe lightbox from map clicks

**Details:**
- Photo markers replaced from 10px cyan dots to 48x48px thumbnail divIcons
- Programmatic PhotoSwipe lightbox with dataSource from all 55 route photos
- Per-marker click calls lightbox.loadAndOpen(index) — no popup intermediate
- showHideAnimationType: 'fade' (no DOM anchor for zoom)
- lightbox.pswp?.close() wired to map:reset event

# Milestone v5.0: Strava Integration + Results

**Status:** ✅ SHIPPED 2026-03-30
**Phases:** 27-32
**Total Plans:** 10

## Overview

Integrate Strava segment data into the existing sector and KOM cards, build a Grinduro-style dual scoring engine (cumulative time for gravel, top-10 points for KOM/QOM), create a Strava OAuth activity submission flow on Netlify Functions with per-athlete JSON storage, and ship a results page with gravel champion and KOM/QOM champion leaderboards. Pre-event features (segment links, scoring explainer, manual KOM times) ship first with zero API dependency, while the Strava app registration begins immediately to clear its 2-4 week review window.

## Milestone Summary

**Key Decisions:**
- Netlify Functions v1 (exports.handler) — active v2 env var bug as of 2026-03-28
- OAuth state = base64url JSON — activity URL survives round-trip without server storage
- activity:read_all scope (not activity:read) for include_all_efforts=true
- Gender from form (M/F/NB), not Strava profile — schema intent + self-identification
- String segment IDs throughout scoring engine — Strava returns integer, String() cast required
- Tie-safe dense ranking — athletes with identical times share rank and points
- Combined genders per segment leaderboard — avoids 27 mini-boards
- Root-cause fix in resolve-annotations.js, not manual JSON — ensures idempotency

**Issues Resolved:**
- Prebuild pipeline clobbering Strava fields (Phase 32 gap closure)
- Segment time tie handling non-deterministic ranking (Phase 28 auto-fix)

**Issues Deferred:**
- GENDERS constant exported from scoring.js but unused by external consumers (minor tech debt)

**Technical Debt Incurred:**
- GENDERS constant in scoring.js unused externally — validate-results.mjs defines own VALID_GENDERS inline
